# 동글이의 모험 — 설계 문서

- **작성일**: 2026-05-03
- **상태**: 승인 대기 (사용자 검토 중)
- **대상 산출물**: Python + Pygame 데스크톱 게임

> English version: [2026-05-03-dongles-adventure-design.en.md](./2026-05-03-dongles-adventure-design.en.md)

## 1. 컨셉

흰 페르시안 고양이 **동글이**가 무한히 솟아오르는 캣타워를 점프로 등반하는 세로 스크롤 플랫포머. MSX 클래식 *요술나무(Magical Tree, 코나미 1984)* 의 등반/추락사 메커니즘에 무한 절차 생성을 결합.

### 1.1 컨셉 한 줄 요약
"요술나무의 점프 등반 + Doodle Jump식 무한 진행 + 흰 고양이 테마"

### 1.2 디자인 핵심 가치
- **짧은 세션, 즉시 재시작**: 한 판 1~5분, 죽으면 한 키로 다시
- **명확한 진행감**: 높이가 곧 점수
- **환경의 점진적 변화**: 위로 갈수록 위험·시각·청각 모두 변화

## 2. 핵심 게임플레이

### 2.1 목표
가능한 한 높이 올라가 **최고 높이 기록**을 갱신.

### 2.2 조작
- `←` `→` : 좌우 이동
- `Space` : 점프 (탭 = 짧은 점프, 길게 누름 = 높은 점프 — 가변 점프 높이)
- `Esc` : 일시정지
- `R` : (게임오버 시) 재시작

### 2.3 카메라
플레이어를 화면 중하단(약 60% 위치)에 유지하는 부드러운 세로 스크롤. 카메라는 도달한 **최대 높이**를 추적하며 절대 아래로 내려오지 않음(Doodle Jump식). 따라서 한 번 올라갔던 발판이 화면 아래로 사라지면 다시 돌아갈 수 없고, 화면 아래로 추락 시 즉사.

### 2.4 사망 조건
- 화면 아래(카메라 하단) 밖으로 추락
- 적/유해물 접촉 (무적 상태가 아닐 때)

### 2.5 세션 흐름
메뉴 → 게임 → 게임오버(점수 표시) → 재시작 또는 메뉴 복귀.

## 3. 발판 (절차 생성)

### 3.1 발판 종류
| 종류 | 동작 | 등장 시작 높이 |
|---|---|---|
| 일반 캣타워 단 | 안정적, 표준 | 0m |
| 천 해먹 | 안정적, 약간 출렁임 | 0m |
| 로프 | 좁은 발판, 정확한 점프 필요 | 50m |
| 흔들리는 발판 | 좌우로 천천히 움직임 | 100m |
| 사라지는 발판 | 착지 후 1초 뒤 사라짐 | 200m |
| 끈끈이 테이프 | 점프 불가, 곧장 추락 | 300m |

### 3.2 생성 규칙
- 청크 단위로 생성 (한 청크 = 화면 높이 1개분)
- 다음 청크는 플레이어가 현재 청크의 중간을 지날 때 생성
- 한 청크에 발판 4~7개, 수평 간격은 최대 점프 거리의 70%를 넘지 않도록 보장(닿을 수 있는 길이 항상 존재)
- 높이가 올라갈수록:
  - 발판 간 수직 간격 ↑
  - 위험 발판 비율 ↑
  - 일반 발판 비율 ↓

## 4. 위험 요소

### 4.1 높이별 등장표
| 높이대 | 추가되는 위협 |
|---|---|
| 0~50m | (튜토리얼) 떨어지는 털실뭉치 |
| 50~150m | + 쥐 (좌우로 빠르게 달림) |
| 150~300m | + 까마귀 (좌우로 날아다니며 곡선 이동) |
| 300~500m | + 강아지 (특정 발판 위에서 짖으며 점프 차단), 분무기 물줄기 (수평으로 발사) |
| 500m+ | + 청소기 (화면 아래에서 일정 속도로 상승, 닿으면 즉사) |

### 4.2 행동 규칙
- 모든 위협은 화면 안 ±1청크 범위에서만 active (성능)
- 청소기는 500m 도달 순간 등장하며, 플레이어 속도와 관계없이 일정 속도로 따라옴 → 영구 압박

## 5. 아이템 / 파워업

| 아이템 | 효과 | 등장 빈도 |
|---|---|---|
| 참치캔 | 점수 +50 | 잦음 |
| 개다래 | 5초 무적 (적 통과 가능, 추락은 여전히 즉사) | 드묾 |
| 깃털 장난감 | 5초 점프력 ↑ | 보통 |
| 물고기 | 목숨 +1 (최대 3개 보유) | 매우 드묾 |

게임 시작 시 목숨 1개. 적에 닿으면 목숨 -1 + **1.5초 무적(i-frame)** 부여(연쇄 피격 방지), 0 미만이면 게임오버. 추락은 목숨 무관 즉사.

## 6. 해상도 / 화면 레이아웃 / HUD

### 6.1 해상도
- **창 크기**: 1920×1080 (Full HD, 고정 / 풀스크린 둘 다 지원)
- **게임 내부 렌더 해상도**: 360×540 → 2x **정수 스케일**로 720×1080 표시 (픽셀 아트 선명도 보장)
- 내부 좌표계 기준으로 모든 게임 로직 작성 → 윈도우/풀스크린 무관하게 동일 동작

### 6.2 화면 분할
| 영역 | 위치 | 크기 (px) |
|---|---|---|
| 왼쪽 HUD | x=0 | 600×1080 |
| 게임 영역 | x=600 | 720×1080 |
| 오른쪽 HUD | x=1320 | 600×1080 |

### 6.3 왼쪽 HUD (게임 상태)
- **ALTITUDE**: 현재 높이 (큰 폰트, 강조)
- **BEST**: 최고 기록
- **LIVES**: 물고기 아이콘 0~3개
- **POWERUPS**: 활성 파워업 + 남은 시간 (예: "🌿 catnip 4.2s")
- **CONTROLS**: 조작 안내 (← → 이동, Space 점프) — 항시 표시

### 6.4 오른쪽 HUD (진행 정보)
- **ALTITUDE SCALE**: 50m, 150m, 300m, 500m 위협 등장 라인을 수직선으로 표시, 현재 높이를 화살표로 강조 → 다음 도전을 시각적으로 예고
- **NEXT HAZARD**: 다음 등장 예정 위협의 아이콘 + 높이 (예: "🐦 Crows @ 150m")
- **TUNA**: 누적 참치캔 수
- **HINT**: "Esc: Pause", "R: Restart"

### 6.5 게임 화면 내 오버레이
- **일시정지**: 게임 영역 중앙에 반투명 배너 + "PAUSED — Press Esc to resume"
- **게임오버**: 게임 영역 중앙에 큰 결과 패널 (도달 높이, 신기록 여부, "Press R to restart")

## 7. 시각 스타일

- **16비트 픽셀 아트** (BGM 톤과 매칭)
- 동글이 스프라이트: 32×32px
- 애니메이션 상태: idle, run, jump (상승), fall (하강), hurt
- 배경: 캣타워 내부 + 창문으로 보이는 바깥 풍경
  - 0~150m: 낮 (파란 하늘)
  - 150~350m: 노을 (주황)
  - 350~500m: 밤 (어두운 보라)
  - 500m+: 우주 (검은 배경 + 별)

## 8. 사운드

- **BGM**: 흥겨운 16비트 칩튠 풍 (개발 중엔 placeholder 트랙 사용, 마무리 단계에 본 BGM 결정 — D안)
- **효과음**: 점프, 착지, 아이템 획득, 피격, 게임오버 (placeholder도 OK)
- Pygame `mixer` 모듈 사용, OGG 권장

## 9. 데이터 영속화

- 최고 점수만 로컬 저장 (`data/highscore.json`)
- 형식: `{"high_score_m": <int>}`
- 게임오버 시 갱신 시 즉시 저장

## 10. 기술 구조

### 10.1 스택
- Python 3.11+
- Pygame 2.5+
- 의존성 관리: `pyproject.toml` (formula: 형제 프로젝트 galaga-clone과 동일)

### 10.2 디렉토리 구조
```
dongles-adventure/
├── main.py                   # 진입점
├── pyproject.toml
├── README.md
├── settings.py               # 화면 크기, FPS, 색상, 키바인딩, 물리 상수
├── assets/
│   ├── sprites/              # 동글이, 적, 발판, 아이템, 배경
│   ├── sounds/               # 효과음
│   └── music/                # BGM (placeholder)
├── engine/
│   ├── __init__.py
│   ├── camera.py             # 세로 스크롤 카메라
│   ├── physics.py            # 중력, 점프, 충돌
│   └── input.py              # 키 입력 처리
├── entities/
│   ├── __init__.py
│   ├── player.py             # 동글이
│   ├── platforms.py          # 발판 종류별 클래스
│   ├── hazards.py            # 적/유해물 (털실, 쥐, 까마귀, 강아지, 분무기, 청소기)
│   └── items.py              # 아이템 (참치캔, 개다래, 깃털, 물고기)
├── world/
│   ├── __init__.py
│   ├── generator.py          # 절차적 청크 생성
│   └── difficulty.py         # 높이별 난이도 곡선
├── scenes/
│   ├── __init__.py
│   ├── menu.py
│   ├── game.py
│   └── gameover.py
├── data/
│   └── highscore.json
└── tests/
    ├── test_physics.py
    ├── test_generator.py
    └── test_difficulty.py
```

### 10.3 주요 모듈 책임
- **engine/physics.py**: 중력, 점프 임펄스, AABB 충돌. 상태 없는 순수 함수 위주.
- **engine/camera.py**: 월드 좌표 ↔ 화면 좌표 변환. 카메라는 `y_top`만 추적.
- **world/generator.py**: 청크 생성 / 폐기. `generate_chunk(y_start, difficulty) -> Chunk`.
- **world/difficulty.py**: 높이 → 난이도 파라미터 매핑(테이블 기반).
- **entities/\***: 각각 `update(dt, world)` 와 `draw(screen, camera)` 인터페이스.
- **scenes/\***: scene stack 패턴. 각 scene은 `handle_event`, `update`, `draw`.

## 11. 테스트 전략

- 단위 테스트(pytest): physics(중력, 충돌), generator(청크 보장 — "도달 가능한 발판이 항상 존재"), difficulty 매핑.
- Pygame 의존(렌더링) 부분은 직접 테스트하지 않음 — 수동 플레이 테스트로 검증.
- 절차 생성기는 시드 고정으로 결정론적 테스트.

## 12. 개발 단계 (이후 implementation plan에서 세분화)

1. 프로젝트 스캐폴딩 (pyproject, 디렉토리, Pygame 윈도우)
2. 동글이 + 정적 발판 + 점프 + 충돌
3. 카메라 + 추락사 + 게임오버 + 재시작
4. 절차적 발판 생성 + 청크 관리
5. 위협 타입 단계적 추가 (1종씩, 높이 게이팅 포함)
6. 아이템 + HUD + 목숨 시스템
7. 메뉴 / 최고점수 저장
8. 사운드 / BGM 연결 (placeholder)
9. 시각 폴리싱 (애니메이션, 배경 변화, 이펙트)
